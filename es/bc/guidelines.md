# Product Guidelines

Estas guidelines definen los principios de diseño, branding y estándares de accesibilidad para la aplicación Anti-Money Laundering (AML), asegurando una experiencia de usuario consistente y efectiva.

## Principios UI/UX

### Alta densidad de datos
La interfaz debe estar optimizada para mostrar grandes volúmenes de datos tabulares, logs e historiales de transacciones sin sobrecargar al usuario.

### Accionabilidad clara
Las acciones clave (ej. bloquear transacciones, aprobar alertas, definir políticas) deben ser visualmente distintas e inmediatamente accesibles para reducir la carga cognitiva durante revisiones críticas.

### Minimalista y limpio
Reducir el ruido visual y elementos innecesarios para ayudar a los analistas a enfocarse estrictamente en indicadores de riesgo críticos y datos de compliance.

## Branding & Tono

### Accesible y claro
El copy de la aplicación debe priorizar un lenguaje claro y sencillo. Las políticas AML complejas, conceptos zero-trust y verificaciones criptográficas deben explicarse de forma fácilmente comprensible por oficiales de compliance no técnicos.

## Estética Visual

::: tip Corporate Enterprise
El diseño debe adoptar una estética enterprise tradicional. Esto incluye usar una paleta de **azules apagados**, **grises**, y mantener **alto contraste** para asegurar legibilidad y una apariencia profesional adecuada para instituciones financieras.
:::

## Accesibilidad & Internacionalización (i18n)

| Requisito | Detalle |
|-----------|---------|
| **WCAG 2.1 AA** | Cumplimiento estricto de estándares de accesibilidad en todas las interfaces |
| **Soporte multilingüe** | Diseñar componentes UI con i18n en mente, acomodando diferentes longitudes de texto y direcciones |
| **Screen Reader** | Proporcionar aria-labels claros, orden de tabulación lógico y navegación por teclado robusta, especialmente para data grids complejos y dashboards interactivos |
